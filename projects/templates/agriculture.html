Coding Part

NodeMcu

 Our Microcontroller is NodeMcu ESP8266 with a temperature and humidity sensor (DHT11) and a moisture sensor. These components have an ADC converter to connect them with the microcontroller, and they are all connected to the MCU via the GPIO data pins.

 We used PlatformIO IDE instead of Arduino IDE because PIO has extension support in VSCode. It provides all the tools needed to build and upload the code to the embedded system, with Adafruit board configuration and header files to support LCD display, making the NodeMCU act as a web server, CPP JSON library, etc.

 The HTTP POST request is made every 5 seconds from the NodeMCU to the server, which is JSON of the field parameters along with the motor status, which is a boolean value. When the server accesses the specific endpoint in the NodeMCU server with the search param ?motor=on/off, it processes the param and works accordingly.

Step to copy the code

  1. Clone the Repo or download the zipped file
  2. Install PIO Extention in VSCode and add the respective headfiles
  3. Check if COM Driver is installed in the PC
  4. Build and Upload the code in NodeMcu

python - Flask

 The Centralised web server is scripted with Flask - python's micro web framwork, with is responsible to process the data flow from prototype to the mobile, it accepts and display the recived data in the debug terminal the recieved data is accessible in the endpoint "/sensor-readings", where the data is constantly updated from the recieved data.

Step to copy the code

  1. Clone the Repo or downloaded the zipped file
  2. Install python libraries and flask framework with pip
  3. Run python app.py

python - Machine Learning

 This part of the projects contains compiled machine learning model where the trained model is in .pkl format, by using scikit-learn library and joblib package, we can query the input parameters to the model and get the predicted label.

  1. This folder contains jupyter Notebook files that have code to train the model
  2. The dataset that we used to train and test the model
  3. Notebook file to visulize the data and prediction accuracy of the models
